# 專案行為規範

## 回覆語言

所有回覆與程式碼註解一律使用**繁體中文**。

---

## TMDB 使用規範

本專案使用 TMDB，所有設計與實作**必須同時符合**：

- `TMDB_TERMS.txt`
- `TMDB_API_TERMS.txt`

提供 TMDB 相關建議前，必須對照上述兩份文件，不得臆測條款內容。
若可能違反任一文件，必須：

1. 明確指出風險
2. 標示對應的條款標題或段落
3. 停止提供該違規作法

---

## 工具與服務選用原則

優先使用免費、開源，或有明確免費方案的工具

若建議付費方案，必須說明：

1. 費用模式與是否有免費額度
2. 為何無法使用免費方案
3. 至少提供一種可行的免費替代方案

---

## 工作檢查清單

每次有實質變更後，執行：

```bash
npm run lint
```

並修正所有錯誤後再提交。

---

## Commit 建議訊息

當使用者詢問 commit 時：

1. **先執行**：

```bash
git status -sb
```

2. 依據變更檔案內容，提供對應的**繁體中文 commit 建議訊息**。
